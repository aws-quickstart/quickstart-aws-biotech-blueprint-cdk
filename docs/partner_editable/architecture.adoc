Deploying this Quick Start for a new virtual private cloud (VPC) with default parameters builds the following {partner-product-short-name} environment in the AWS Cloud.

:xrefstyle: short
[#architecture1]
.Quick Start architecture for {partner-product-short-name} on AWS
image::../images/biotech-blueprint-architecture-diagram.png[Architecture,width=100%,height=100%]

As shown in <<architecture1>>, the Quick Start sets up the following:

* A highly available architecture that spans two Availability Zones.
* Preclinical production and development virtual private clouds (VPCs) with public and private subnets according to AWS best practices, to provide you with your own virtual network on AWS.
* A management VPC with AWS Client VPN endpoints in the public subnets.
* Peering connections to allow Secure Shell (SSH) or remote desktop access from the management VPC to private subnets in the production and research VPCs.
* AWS Config to assess the overall compliance of your AWS resources with the following conformance packs:
** AWS Control Tower Detective Guard Rails Conformance Pack.
** Operational Best Practices for AWS Identity and Access Management (IAM).
** Operational Best Practices for NIST Cyber Security Framework (CSF).
** Operational Best Practices for Amazon Simple Storage Service (Amazon S3).
* Amazon Route 53 for a private Domain Name System (DNS).
* An optional AWS Service Catalog portfolio with the following informatics applications:
** ChemAxon Compound Registration
** Nextflow
** Dotmatics
** Hail 0.2
** Titian Mosaic FreezerManagement
