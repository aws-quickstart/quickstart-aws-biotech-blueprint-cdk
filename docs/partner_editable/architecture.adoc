Deploying this Quick Start for a new virtual private cloud (VPC) with default parameters builds the following {partner-product-short-name} environment in the AWS Cloud.

:xrefstyle: short
[#architecture1]
.Quick Start architecture for {partner-product-short-name} on AWS
image::../images/biotech-blueprint-architecture-diagram.png[Architecture,width=100%,height=100%]

As shown in <<architecture1>>, the Quick Start sets up the following:

* A highly available architecture that spans two Availability Zones.
* Three virtual private clouds (VPCs) configured with public, private, and isolated subnets, according to AWS best practices, to provide you with your own virtual network on AWS.
** A production VPC into which you can deploy informatics software from the AWS Services Catalog.
** A management VPC with AWS Client VPN in the public subnets to provide secure VPN access using OpenVPN-based clients.
** A research VPC into which you can deploy research software from the AWS Services Catalog.
* Peering connections to allow Secure Shell (SSH) or remote desktop access to private subnets in the production and research VPC from the management VPC.
* AWS Config to continuously audit and assess the overall compliance of your AWS resources with the following conformance packs:
** AWS Control Tower Detective Guard Rails Conformance Pack.
** Operational Best Practices for AWS Identity and Access Management (IAM).
** Operational Best Practices for NIST Cyber Security Framework (CSF).
** Operational Best Practices for Amazon Simple Storage Service (Amazon S3).
* Amazon Route 53 for a private Domain Name System (DNS).
* AWS Service Catalog with the following informatics software that you can deploy into the production VPC whenever you want:
** ChemAxon Compound Registration
** Nextflow
** Dotmatics
** Hail 0.2
** Titian Mosaic FreezerManagement
